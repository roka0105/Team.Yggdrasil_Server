통신할때는 필요한 데이터만 주고 받고 할것이고 
그것에 대한 데이터들은 차곡차곡 서버에 적용되어 있을건데
그렇게 모인 하나의 데이터 그룹을 통째로 받아오는게 필요할때
info class 객체를 사용한다.

예를 들어 roomoutinfo 정보를 클라들이 방을 만들때마다 서버에서
해당 list 정보들을 저장하고 있을건데
그 정보중 클라에서 3page 방 정보들만 보내줘 했을때
방 정보를 받아와서 언팩할때 roomoutinfo 를 사용한다.


오늘 진행된 사항
멀티 누를 시 로비 입장
사용할 정보 클래스 만듦 roomoutinfo 
(그 외에 player,character,roominfo 등 있지만 변경사항이 생길 수 있고
당장 쓰지 않기 때문에 틀만 만들어 둠)

해야할 것
1.lobby 의 버튼 입력시 적용되는 클라 동작들 모두 구현 후 연결 시켜 놓기.
2.lobby 관련 서버랑 통신 하는 부분 구현하기.
  (
   1. 방 리스트 받아와서 세팅하는것
   2.방 만드는것
   3.채팅
  )
